# Expectation-Maximization

Expectation-Maximization (EM) is an algorithm for maximum likelihood estimation when the model contains hidden, missing, or latent variables.

The core problem is this:

> If we knew the hidden variables, estimating the parameters would be easy. But we do not know them.

EM solves this by alternating between two steps:

- **E-step**: estimate the hidden variables using the current parameters.
- **M-step**: update the parameters using those estimated hidden variables.

This turns a hard optimization problem into a sequence of easier ones.

## Intuition: soft labels for hidden structure

Imagine a dataset that was generated by two Gaussian clusters, but the cluster labels are hidden. If we knew which point came from which cluster, estimating each mean and variance would be straightforward.

But the labels are not observed. EM handles this by assigning each point a soft responsibility:

$$
r_{ik} = P(z_i = k \mid x_i, \theta)
$$

This value says how responsible component $k$ is for data point $x_i$.

The algorithm then alternates:

1. Use current clusters to compute responsibilities.
2. Use responsibilities to refit the clusters.
3. Repeat until the parameters stop changing.

## The latent-variable likelihood

Let $x_i$ be an observed data point and $z_i$ be a latent variable. For example, in a Gaussian Mixture Model, $z_i$ is the unknown cluster identity.

The likelihood of one observation is

$$
P(x_i \mid \theta)
=
\sum_{z_i} P(x_i, z_i \mid \theta)
$$

For the full dataset:

$$
\mathcal{L}(\theta)
=
\prod_{i=1}^{n}
\sum_{z_i} P(x_i, z_i \mid \theta)
$$

The log-likelihood is

$$
\ell(\theta)
=
\sum_{i=1}^{n}
\log
\sum_{z_i} P(x_i, z_i \mid \theta)
$$

The difficulty is the log of a sum. If the latent variables were known, the complete-data log-likelihood would usually be much easier:

$$
\log P(\mathcal{D}, \mathcal{Z} \mid \theta)
$$

EM works by optimizing the expected complete-data log-likelihood instead.

## The EM algorithm

Suppose the current parameter estimate is $\theta^{(t)}$.

### E-step

Compute the posterior distribution over latent variables using the current parameters:

$$
P(\mathcal{Z} \mid \mathcal{D}, \theta^{(t)})
$$

Then form the expected complete-data log-likelihood:

$$
Q(\theta \mid \theta^{(t)})
=
\mathbb{E}_{\mathcal{Z} \mid \mathcal{D}, \theta^{(t)}}
\left[
\log P(\mathcal{D}, \mathcal{Z} \mid \theta)
\right]
$$

### M-step

Choose parameters that maximize this expectation:

$$
\theta^{(t+1)}
=
\arg\max_{\theta}
Q(\theta \mid \theta^{(t)})
$$

Then repeat until convergence.

## Why EM improves the likelihood

EM can be understood through a lower bound on the log-likelihood. The E-step constructs a bound that touches the current log-likelihood. The M-step improves that bound.

As a result, each EM iteration does not decrease the observed-data log-likelihood:

$$
\ell(\theta^{(t+1)}) \ge \ell(\theta^{(t)})
$$

This monotonic improvement is one of the main reasons EM is useful. However, it does not guarantee finding the global maximum. EM can converge to a local maximum or a saddle point, so initialization matters.

## EM for Gaussian Mixture Models

A Gaussian Mixture Model (GMM) assumes that each observation comes from one of $K$ Gaussian components:

$$
P(x_i \mid \theta)
=
\sum_{k=1}^{K}
\pi_k \mathcal{N}(x_i \mid \mu_k, \Sigma_k)
$$

where

- $\pi_k$ is the mixture weight for component $k$.
- $\mu_k$ is the mean of component $k$.
- $\Sigma_k$ is the covariance of component $k$.
- $z_i$ is the hidden component assignment.

### E-step for GMMs

Compute the responsibility of component $k$ for point $x_i$:

$$
r_{ik}
=
P(z_i=k \mid x_i, \theta)
=
\frac{
\pi_k \mathcal{N}(x_i \mid \mu_k, \Sigma_k)
}{
\sum_{j=1}^{K}
\pi_j \mathcal{N}(x_i \mid \mu_j, \Sigma_j)
}
$$

### M-step for GMMs

Let

$$
N_k = \sum_{i=1}^{n} r_{ik}
$$

Then update

$$
\pi_k^{\text{new}} = \frac{N_k}{n}
$$

$$
\mu_k^{\text{new}}
=
\frac{1}{N_k}
\sum_{i=1}^{n}
r_{ik}x_i
$$

$$
\Sigma_k^{\text{new}}
=
\frac{1}{N_k}
\sum_{i=1}^{n}
r_{ik}(x_i-\mu_k^{\text{new}})(x_i-\mu_k^{\text{new}})^T
$$

These updates are weighted versions of the usual sample estimates. The weights are the responsibilities.

## EM vs k-means

K-means can be seen as a hard-assignment cousin of EM for Gaussian mixtures with strong simplifying assumptions.

| Idea | K-means | EM for GMM |
| --- | --- | --- |
| Assignment | Hard cluster label | Soft responsibility |
| Cluster shape | Spherical, similar size | Can model different covariances |
| Objective | Minimize squared distances | Maximize likelihood |
| Uncertainty | Not represented | Represented through responsibilities |

When clusters overlap, soft assignments often give a more realistic description than hard labels.

## Practical details

EM is conceptually clean, but several details matter in practice:

- **Initialization matters**: different starting points can lead to different local optima.
- **Likelihood should be monitored**: it should not decrease between iterations.
- **Convergence can be slow**: EM often makes small improvements near the optimum.
- **Degenerate solutions can occur**: in GMMs, a component can collapse onto one point unless covariances are regularized.
- **Model selection is separate**: EM estimates parameters for a chosen number of components; it does not automatically choose $K$.

## Summary table

| Question | Answer |
| --- | --- |
| When do we use EM? | When likelihood contains hidden or missing variables |
| What happens in the E-step? | Compute the posterior over latent variables using current parameters |
| What happens in the M-step? | Maximize the expected complete-data log-likelihood |
| Does EM always improve likelihood? | It should not decrease the observed-data log-likelihood |
| Does EM find the global optimum? | Not necessarily; it can converge to a local optimum |
| Common example | Gaussian Mixture Models |

## Where these ideas reappear

- **Gaussian Mixture Models**: EM is the standard fitting algorithm.
- **Hidden Markov Models**: Baum-Welch is an EM algorithm for HMMs.
- **Missing-data problems**: EM can estimate parameters when part of the data is unobserved.
- **Probabilistic PCA and factor analysis**: latent continuous variables can be handled with EM-style updates.
- **Topic models and mixture models**: many latent-variable models use EM or variational EM.
- **Variational inference**: coordinate-ascent variational inference generalizes the same lower-bound idea.
